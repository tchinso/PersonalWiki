# 위키 문법 설명서

PersonalWiki 문서는 Markdown을 기본으로 쓰고, 문서 링크, 템플릿, 이미지 단축 문법, 목차, 콜아웃 같은 전용 문법을 추가로 지원합니다.

이 문서의 예시는 대부분 코드 블록 안에 들어 있습니다. 실제 문서에 사용할 때는 코드 블록 바깥에 붙여 넣으면 됩니다.

## 빠른 예시

```markdown
# 문서 제목

[[다른 문서]]로 이동하거나 [[다른 문서|표시 이름]]을 따로 줄 수 있습니다.

==중요한 문장==은 형광펜처럼 보이고, ||숨김 텍스트||는 클릭해서 열어봅니다.

!!!info 도움말
!!!info F1 : 도움말
!!!info F2 : 이름 변경
```

## 기본 Markdown

### 제목

```markdown
# 1단계 제목
## 2단계 제목
### 3단계 제목
#### 4단계 제목
```

제목에는 자동으로 앵커가 붙습니다. 같은 제목이 여러 번 나오면 뒤에 번호가 붙습니다.

### 문단과 줄바꿈

```markdown
문단은 빈 줄로 나눕니다.

이렇게 한 줄을 비우면 새 문단입니다.
```

표 셀 안에서 줄을 바꾸고 싶을 때는 `<br>`을 사용합니다.

```markdown
| 이름 | 설명 |
| --- | --- |
| Yunli | 첫 줄<br>둘째 줄 |
```

### 강조

```markdown
*기울임*
**굵게**
~~취소선~~
`인라인 코드`
```

PersonalWiki 전용 강조도 사용할 수 있습니다.

```markdown
==형광펜==
||스포일러||
```

### 목록

```markdown
- 항목
- 항목
  - 하위 항목

1. 첫 번째
2. 두 번째

- [ ] 할 일
- [x] 끝난 일
```

### 인용문

```markdown
> 인용문
>> 중첩 인용문
```

### 코드 블록

````markdown
```python
print("hello")
```
````

### 가로선

```markdown
---
```

## 링크

### 일반 링크

```markdown
[OpenAI](https://openai.com)
https://example.com
```

URL을 그대로 적어도 자동 링크로 렌더링됩니다.

### 문서 링크

```markdown
[[문서명]]
[[문서명|표시 텍스트]]
```

문서가 있으면 해당 문서로 이동하고, 없으면 새 문서 작성 화면으로 이동합니다.

`img/`, `file/`, `http://`, `https://`, `TOC`, `TOC1`부터 `TOC6`까지는 내장 문법과 충돌할 수 있으므로 문서 제목으로 쓰지 않는 편이 안전합니다.

### 파일 링크

```markdown
[[file/report.pdf]]
[[file/folder/archive.zip]]
```

파일은 실행 파일 옆의 `file` 폴더에 직접 넣습니다.

## 이미지와 동영상

### Markdown 이미지

```markdown
![대체 텍스트](https://example.com/image.png)
```

### PersonalWiki 이미지 단축 문법

```markdown
![[strawberry.webp]]
![[strawberry.webp|딸기]]
![[strawberry.webp, width=640]]
![[strawberry.webp, height=360]]
![[strawberry.webp|딸기, width=640, height=360]]
```

이미지는 실행 파일 옆의 `img` 폴더에 직접 넣습니다. `width`와 `height`는 120부터 3840 사이로 제한됩니다.

### YouTube 임베드

```markdown
![[youtube(HhnETSN6U_E)]]
![[youtube(HhnETSN6U_E, width=640, height=360)]]
![[youtube(HhnETSN6U_E, start=90)]]
```

`start`에는 초 단위 숫자 또는 `1:30` 같은 시간을 넣을 수 있습니다.

## 표

PersonalWiki는 Mistune의 표 문법을 기반으로 하며, 일반 Markdown 표와 열 비율 문법을 함께 지원합니다.

### 기본 표

```markdown
| 이름 | 설명 |
| --- | --- |
| Yunli | 캐릭터 |
| March | 캐릭터 |
```

앞뒤 파이프를 생략한 표도 사용할 수 있습니다.

```markdown
이름 | 설명
--- | ---
Yunli | 캐릭터
March | 캐릭터
```

### 정렬

구분 행에 콜론을 붙이면 정렬을 지정합니다.

```markdown
| 왼쪽 | 가운데 | 오른쪽 |
| :--- | :---: | ---: |
| A | B | C |
```

### 열 비율

구분 행의 `-` 개수가 열의 상대 비율이 됩니다.

```markdown
| 좁게 | 넓게 |
| --- | ------ |
| 1 | 2 |
```

위 표는 첫 번째 열과 두 번째 열의 비율이 `3:6`, 즉 `1:2`가 됩니다. 콜론 정렬과 같이 써도 됩니다.

```markdown
| 왼쪽 | 오른쪽 |
| :--- | ------: |
| A | B |
```

### 표 중간에서 열 개수 바꾸기

본문 중간에 새 구분 행을 넣으면 그 아래 행부터 열 개수와 비율이 바뀝니다. 구분 행 자체는 화면에 표시되지 않습니다.

```markdown
| 운리 |
| --- |
| Yunli |
| --- | ------ |
| 나이 | 20살 |
| 성별 | 여자 |
```

첫 구간은 1열이고, 두 번째 구분 행 아래부터는 2열입니다. 1열 행은 전체 폭을 차지하고, 2열 행은 `3:6` 비율을 사용합니다.

구분 행을 다시 넣으면 계속 바꿀 수 있습니다.

```markdown
| 캐릭터 |
| --- |
| Yunli |
| --- | ------ |
| 나이 | 20살 |
| 성별 | 여자 |
| --- | --- | --- |
| 속성 | 물리 | 파멸 |
```

주의할 점:

- 첫 번째 행과 바로 아래 구분 행의 열 수는 같아야 합니다.
- 새 구분 행 뒤의 일반 행은 다음 구분 행이 나오기 전까지 그 구분 행의 열 수를 따라야 합니다.
- 표 안에서 줄바꿈이 필요하면 `<br>`을 사용합니다.
- 편집 화면의 표 편집기는 행별 열 수와 `---` 길이를 편집할 수 있고, 필요한 구분 행을 자동으로 생성합니다.

## 콜아웃

콜아웃은 문서 안에서 안내, 메모, 경고를 강조하는 상자입니다.

```markdown
!!!note 메모
!!!info 정보
!!!warn 경고
!!!danger 위험
```

띄어쓰기는 다음처럼 써도 됩니다.

```markdown
!!! info 정보
```

### 같은 유형 연속 콜아웃 묶기

같은 유형의 콜아웃이 빈 줄 없이 연속으로 나오면 하나의 상자로 묶입니다. 아이콘은 첫 줄에 한 번만 표시됩니다.

```markdown
!!!info 도움말
!!!info F1 : 도움말
!!!info F2 : 이름 변경
```

다음 경우에는 묶이지 않습니다.

```markdown
!!!info 도움말
F1 : 도움말
F2 : 이름 변경
```

콜아웃 줄이 아닌 일반 문장이 중간에 있으므로 하나의 콜아웃으로 합쳐지지 않습니다.

```markdown
!!!info 도움말
!!!note F1 : 도움말
```

유형이 달라졌으므로 서로 다른 콜아웃으로 렌더링됩니다.

## 각주

```markdown
본문 중간에 각주를 넣습니다[^1].

[^1]: 이것이 각주 내용입니다.
```

각주는 호출하는 문장과 각주 정의가 모두 있어야 정상적으로 표시됩니다. 각주 정의 앞뒤로 빈 줄을 두면 가장 안정적입니다.

## 템플릿

다른 문서의 내용을 현재 문서 안에 가져올 수 있습니다.

```markdown
{{공통문서}}
```

템플릿으로 불러온 문서 안의 템플릿은 다시 확장되지 않습니다. 순환 참조와 과한 중첩을 막기 위한 제한입니다.

접힌 템플릿은 다음처럼 씁니다.

```markdown
||{{공통문서}}||
```

## 목차

목차는 현재 문서의 제목을 모아서 만듭니다.

```markdown
{{TOC}}
```

기본값은 2단계 제목까지입니다. 숫자를 붙이면 표시할 제목 깊이를 정합니다.

```markdown
{{TOC1}}
{{TOC3}}
{{TOC6}}
```

`{{TOC3}}`은 `#`, `##`, `###` 제목까지 표시합니다.

## HTML

일부 HTML은 그대로 사용할 수 있습니다.

```markdown
<kbd>Ctrl</kbd> + <kbd>S</kbd>
```

HTML은 문서를 깔끔하게 만들 때만 제한적으로 쓰는 편이 좋습니다. 표 안 줄바꿈에는 `<br>`이 가장 흔하게 쓰입니다.

## 검색

검색창에서는 기본 단어 검색과 함께 `AND`, `OR`, `NOT` 연산자를 사용할 수 있습니다.

```text
flask AND sqlite
python NOT django
template OR snippet
```

문서 제목, 본문, 태그가 검색 대상입니다.

## 문서 내보내기

문서 내보내기는 일반 보기와 같은 Markdown 렌더러를 사용합니다. 표의 중간 열 변경, `---` 개수 기반 열 비율, 같은 유형 콜아웃 묶기, 이미지 단축 문법, 템플릿 렌더링 결과가 내보낸 HTML에도 반영됩니다.

단일 HTML로 내보낼 때 `/img` 이미지는 가능한 경우 data URI로 포함됩니다. `/file` 링크는 파일 경로 링크로 남으므로, 첨부 파일까지 함께 전달하려면 ZIP 내보내기를 사용합니다.

## 자주 막히는 부분

- 표가 통째로 문단처럼 보이면 첫 행과 구분 행의 열 수를 확인합니다.
- 표 중간에서 열 수를 바꾸려면 일반 행을 바로 쓰지 말고 먼저 `| --- | --- |` 같은 구분 행을 넣습니다.
- 같은 콜아웃을 묶고 싶으면 줄 사이에 빈 줄을 넣지 않습니다.
- 다른 유형의 콜아웃은 자동으로 분리됩니다.
- 문서 링크가 새 문서 작성으로 이동하면 아직 해당 제목의 문서가 없는 것입니다.
- 이미지가 보이지 않으면 파일 이름, 확장자, `img` 폴더 위치를 확인합니다.
